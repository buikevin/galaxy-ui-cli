The Galaxy UI CLI makes it easy to add components to your project with a single command.

## Commands

### `init`

Initialize your project for Galaxy UI. This sets up the necessary configuration and utilities.

```bash
npx galaxy-ui-cli@latest init
```

#### What it does:

1. Detects your framework (Angular, React, or Vue)
2. Checks for Tailwind CSS installation
3. Creates a `components.json` configuration file
4. Sets up utility functions (like the `cn()` helper)
5. Configures path aliases if needed

#### Options:

```bash
npx galaxy-ui-cli@latest init [options]
```

| Option | Description |
|--------|-------------|
| `-y, --yes` | Skip all prompts and use defaults |
| `--force` | Overwrite existing configuration |

#### Example:

```bash
# Interactive mode (recommended for first-time setup)
npx galaxy-ui-cli@latest init

# Silent mode with defaults
npx galaxy-ui-cli@latest init -y
```

---

### `add`

Add one or more components to your project.

```bash
npx galaxy-ui-cli@latest add [components...]
```

#### Examples:

```bash
# Add a single component
npx galaxy-ui-cli@latest add button

# Add multiple components
npx galaxy-ui-cli@latest add button input checkbox

# Add all form components
npx galaxy-ui-cli@latest add input checkbox select

# Interactive mode - choose from a list
npx galaxy-ui-cli@latest add
```

#### What it does:

1. Reads your `components.json` configuration
2. Fetches the component from the registry
3. Resolves any dependencies
4. Copies files to your project
5. Transforms imports to match your path aliases

#### Options:

| Option | Description |
|--------|-------------|
| `-y, --yes` | Skip confirmation prompts |
| `-o, --overwrite` | Overwrite existing files |
| `--path <path>` | Custom output path |

#### Example with options:

```bash
# Add button without confirmation
npx galaxy-ui-cli@latest add button -y

# Overwrite existing button component
npx galaxy-ui-cli@latest add button --overwrite

# Add to custom path
npx galaxy-ui-cli@latest add button --path src/ui/components
```

## Component Registry

Components are fetched from the Galaxy UI registry. You can see available components:

```bash
# List all available components
npx galaxy-ui-cli@latest add

# This will show an interactive menu
```

### Available Components:

| Component | Description | Dependencies |
|-----------|-------------|--------------|
| `button` | Button component with multiple variants | `cn` utility |
| `input` | Input field with label and validation | `cn` utility |
| `checkbox` | Checkbox with label and indeterminate state | `cn` utility |
| `select` | Dropdown select with search | `cn` utility |
| `calendar` | Date picker calendar | `cn` utility |

More components coming soon!

## Dependency Management

Some components depend on utilities or other components. The CLI automatically handles this:

```bash
# Add input component
npx galaxy-ui-cli@latest add input

# Output:
✓ Found components.json
✓ Fetching component: input
ℹ This component requires: cn utility
? Install dependencies? (Y/n) yes
✓ Installing cn utility
✓ Copying input component
✓ Component installed at src/components/ui/input/
```

## Configuration File

The `components.json` file controls CLI behavior:

```json
{
  "$schema": "https://galaxy-ui.dev/schema.json",
  "framework": "angular",
  "style": "default",
  "tailwind": {
    "config": "tailwind.config.js",
    "css": "src/styles.css",
    "baseColor": "slate"
  },
  "aliases": {
    "components": "src/components",
    "utils": "src/lib/utils"
  }
}
```

### Framework Detection

The CLI automatically detects your framework by checking:

- **Angular**: Looks for `angular.json`
- **React**: Looks for `react` in `package.json` dependencies
- **Vue**: Looks for `vue` in `package.json` or `vite.config.ts`

You can manually override this in `components.json`.

### Path Aliases

The CLI transforms imports based on your aliases:

```typescript
// If aliases.components = "src/components"
// And aliases.utils = "src/lib/utils"

// Component imports become:
import { cn } from '@/lib/utils';
import { ButtonComponent } from '@/components/ui/button';
```

### Adding Multiple Components

```bash
# Add all form components at once
npx galaxy-ui-cli@latest add input checkbox select -y

# Add entire set
npx galaxy-ui-cli@latest add button input checkbox select calendar -y
```

### Updating Components

When components are updated in the registry:

```bash
# Check what's changed (coming soon)
npx galaxy-ui-cli@latest diff button

# Update component
npx galaxy-ui-cli@latest add button --overwrite
```

## Advanced Usage

### Custom Component Location

Override the default component location:

```bash
npx galaxy-ui-cli@latest add button --path src/shared/ui
```

This will place the button in `src/shared/ui/button/` instead of the default location.

### Batch Operations

Create a script to set up all components:

```json
// package.json
{
  "scripts": {
    "ui:setup": "galaxy-ui-cli init -y && galaxy-ui-cli add button input checkbox select calendar -y"
  }
}
```

Then run:

```bash
npm run ui:setup
```

### CI/CD Integration

Use the CLI in your CI/CD pipeline:

```yaml
# .github/workflows/setup.yml
- name: Setup Galaxy UI
  run: |
    npx galaxy-ui-cli@latest init -y
    npx galaxy-ui-cli@latest add button input -y
```

## Troubleshooting

### "components.json not found"

Run `galaxy-ui-cli init` first to create the configuration file.

### "Failed to fetch component"

Check your internet connection. The CLI needs to connect to the registry.

### "Component already exists"

Use the `--overwrite` flag to replace existing components:

```bash
npx galaxy-ui-cli@latest add button --overwrite
```

### Path alias issues

Make sure your `tsconfig.json` has the correct paths configured:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": {
      "@/*": ["src/*"]
    }
  }
}
```
