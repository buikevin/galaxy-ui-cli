The **Stepper** component guides users through multi-step processes like forms, wizards, and onboarding flows.

## Features

- Horizontal and vertical orientations
- Linear and non-linear navigation
- Step completion tracking
- Error state support
- Optional steps
- Editable steps
- Custom icons
- Status color themes
- Built-in navigation buttons
- Keyboard accessible

## Installation

```bash
npx galaxy-ui-cli@latest add stepper
```

## Usage

### Basic Stepper

```typescript
import { Component } from '@angular/core';
import { StepperComponent, StepComponent } from '@/components/ui/stepper';

@Component({
  standalone: true,
  imports: [StepperComponent, StepComponent],
  template: `
    <g-stepper>
      <g-step [stepLabel]="'Personal Info'">
        <div class="space-y-4">
          <galaxy-input [inputLabel]="'Full Name'"></galaxy-input>
          <galaxy-input [inputLabel]="'Email'"></galaxy-input>
        </div>
      </g-step>

      <g-step [stepLabel]="'Address'">
        <div class="space-y-4">
          <galaxy-input [inputLabel]="'Street Address'"></galaxy-input>
          <galaxy-input [inputLabel]="'City'"></galaxy-input>
        </div>
      </g-step>

      <g-step [stepLabel]="'Confirmation'">
        <p>Review your information and submit.</p>
      </g-step>
    </g-stepper>
  `
})
```

## Props

### Stepper

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `stepperOrientation` | `'horizontal' \| 'vertical'` | `'horizontal'` | Layout direction |
| `stepperStatus` | `'basic' \| 'primary' \| 'success' \| 'info' \| 'warning' \| 'danger'` | `'primary'` | Color theme |
| `stepperLinear` | `boolean` | `true` | Require sequential completion |
| `stepperSelectedIndex` | `number` | `0` | Active step index |
| `stepperShowButtons` | `boolean` | `true` | Show navigation buttons |

### Step

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `stepLabel` | `string` | `''` | Step name |
| `stepDescription` | `string \| null` | `null` | Additional description |
| `stepIcon` | `string \| null` | `null` | Custom Lucide icon |
| `stepCompleted` | `boolean` | `false` | Mark as completed |
| `stepError` | `boolean` | `false` | Show error state |
| `stepOptional` | `boolean` | `false` | Mark as optional |
| `stepEditable` | `boolean` | `true` | Allow navigation to step |

### Events

| Event | Type | Description |
|-------|------|-------------|
| `stepperSelectionChange` | `EventEmitter<number>` | Emits when active step changes |
| `stepperFinish` | `EventEmitter<void>` | Emits when finish button clicked |

## Examples

### Registration Form

```typescript
@Component({
  template: `
    <g-stepper
      [stepperStatus]="'primary'"
      (stepperSelectionChange)="onStepChange($event)"
      (stepperFinish)="onSubmit()">

      <g-step
        [stepLabel]="'Account'"
        [stepIcon]="'user'"
        [stepCompleted]="accountValid()">
        <div class="space-y-4">
          <galaxy-input
            [inputLabel]="'Email'"
            [(ngModel)]="email"
            [inputType]="'email'">
          </galaxy-input>
          <galaxy-input
            [inputLabel]="'Password'"
            [(ngModel)]="password"
            [inputType]="'password'">
          </galaxy-input>
        </div>
      </g-step>

      <g-step
        [stepLabel]="'Profile'"
        [stepIcon]="'user-circle'"
        [stepCompleted]="profileValid()">
        <div class="space-y-4">
          <galaxy-input
            [inputLabel]="'Full Name'"
            [(ngModel)]="fullName">
          </galaxy-input>
          <galaxy-input
            [inputLabel]="'Company'"
            [(ngModel)]="company"
            [stepOptional]="true">
          </galaxy-input>
        </div>
      </g-step>

      <g-step
        [stepLabel]="'Preferences'"
        [stepIcon]="'settings'"
        [stepOptional]="true">
        <galaxy-checkbox
          [checkboxLabel]="'Email notifications'">
        </galaxy-checkbox>
        <galaxy-checkbox
          [checkboxLabel]="'SMS alerts'">
        </galaxy-checkbox>
      </g-step>

      <g-step
        [stepLabel]="'Review'"
        [stepIcon]="'check-circle'">
        <div class="space-y-2">
          <p><strong>Email:</strong> {{ email }}</p>
          <p><strong>Name:</strong> {{ fullName }}</p>
          <p><strong>Company:</strong> {{ company || 'N/A' }}</p>
        </div>
      </g-step>
    </g-stepper>
  `
})
export class RegistrationForm {
  email = '';
  password = '';
  fullName = '';
  company = '';

  accountValid = computed(() => this.email && this.password);
  profileValid = computed(() => this.fullName);

  onStepChange(index: number) {
    console.log('Step changed to:', index);
  }

  onSubmit() {
    console.log('Form submitted!');
    // Submit form data
  }
}
```

## Best Practices

1. **Step count** - Keep to 3-5 steps for best UX
2. **Clear labels** - Use descriptive step names
3. **Progress feedback** - Show completed steps with checkmarks
4. **Validation** - Validate before allowing next step
5. **Save progress** - Auto-save on step changes
6. **Error handling** - Show errors clearly on relevant step
7. **Optional steps** - Mark optional steps explicitly

## Accessibility

- Use semantic HTML structure
- Support keyboard navigation (Tab, Arrow keys)
- Provide `aria-label` for steps
- Use `aria-current="step"` for active step
- Announce step changes to screen readers
- Maintain focus management between steps

## Keyboard Shortcuts

- **Tab** - Navigate through interactive elements
- **Enter/Space** - Activate step button
- **Arrow keys** - Navigate between step headers (non-linear mode)
