The **Tabs** component allows you to organize content into separate views that users can switch between.

## Features

- Horizontal and vertical orientations
- Lucide icons support
- Badge notifications
- Active state management
- Lazy loading support
- Full width option
- Disabled tabs
- Keyboard navigation

## Installation

```bash
npx galaxy-ui-cli@latest add tabs
```

## Usage

### Basic Tabs

```typescript
import { Component } from '@angular/core';
import { TabsComponent, TabComponent } from '@/components/ui/tabs';

@Component({
  standalone: true,
  imports: [TabsComponent, TabComponent],
  template: `
    <g-tabs>
      <g-tab [tabLabel]="'Profile'" [tabActive]="true">
        <p>Profile content here</p>
      </g-tab>
      <g-tab [tabLabel]="'Settings'">
        <p>Settings content here</p>
      </g-tab>
      <g-tab [tabLabel]="'Messages'">
        <p>Messages content here</p>
      </g-tab>
    </g-tabs>
  `
})
```

## Props

### Tabs

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `tabsOrientation` | `'horizontal' \| 'vertical'` | `'horizontal'` | Tab layout direction |
| `tabsStatus` | `'basic' \| 'primary' \| 'success' \| 'info' \| 'warning' \| 'danger'` | `'primary'` | Color theme |
| `tabsFullWidth` | `boolean` | `false` | Stretch tabs to fill width |

### Tab

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `tabLabel` | `string` | `''` | Tab button text |
| `tabIcon` | `string \| null` | `null` | Lucide icon name |
| `tabBadge` | `string \| number \| null` | `null` | Badge content |
| `tabActive` | `boolean` | `false` | Initial active state |
| `tabDisabled` | `boolean` | `false` | Disabled state |
| `tabLazy` | `boolean` | `false` | Load content only when activated |

### Product Details

```typescript
@Component({
  template: `
    <g-tabs>
      <g-tab [tabLabel]="'Description'" [tabActive]="true">
        <div class="p-4">

          </table>
        </div>

      <g-tab [tabLabel]="'Reviews'" [tabIcon]="'star'" [tabBadge]="reviewCount()">
        <div class="p-4">

  qaCount = signal(45);
  productDescription = signal('High-performance laptop...');
}
```

## Best Practices

1. **Limit tab count** - 3-7 tabs is ideal
2. **Use clear labels** - Short, descriptive names
3. **Default selection** - Set one tab as active initially
4. **Icon consistency** - Use icons for all tabs or none
5. **Lazy loading** - For heavy content, enable lazy loading
6. **Keyboard support** - Arrow keys to navigate tabs

## Accessibility

- Use proper ARIA roles (`role="tablist"`, `role="tab"`, `role="tabpanel"`)
- Support keyboard navigation (Arrow keys, Home, End)
- Provide `aria-selected` for active tab
- Use `aria-controls` to link tabs with panels
- Maintain focus management when switching tabs

## Keyboard Shortcuts

- **Arrow Left/Up** - Previous tab
- **Arrow Right/Down** - Next tab
- **Home** - First tab
- **End** - Last tab
- **Enter/Space** - Activate focused tab
