The **Switch** component provides an elegant toggle for boolean on/off states with smooth animations.

**Switch** là component công tắc thanh lịch cho trạng thái bật/tắt boolean với hiệu ứng mượt mà.

## Features | Tính Năng

- **Smooth animation** | **Hiệu ứng mượt mà**: Animated toggle with transition
- **6 status colors** | **6 màu trạng thái**: Basic, primary, success, info, warning, danger
- **Angular Forms integration** | **Tích hợp Angular Forms**: Works with ngModel and Reactive Forms
- **Disabled state** | **Trạng thái vô hiệu hóa**: Prevent user interaction
- **Custom labels** | **Nhãn tùy chỉnh**: Optional labels for on/off states
- **Dark mode** | **Chế độ tối**: Full dark mode support

## Usage | Sử Dụng

### Basic Usage | Sử Dụng Cơ Bản

```typescript
import { Component, signal } from '@angular/core';
import { SwitchComponent } from '@/components/ui/switch';

@Component({
  selector: 'app-example',
  standalone: true,
  imports: [SwitchComponent],
  template: `
    <g-switch [(checked)]="isEnabled">
    </g-switch>

    <p>Status: {{ isEnabled() ? 'Enabled' : 'Disabled' }}</p>
  `
})
export class ExampleComponent {
  isEnabled = signal(false);
}
```

### With Loading State | Với Trạng Thái Đang Tải

```typescript
@Component({
  template: `
    <g-switch
      [(checked)]="isEnabled"
      [disabled]="isLoading()"
      (checkedChange)="handleToggle($event)"
      label="Feature toggle">
    </g-switch>

    @if (isLoading()) {
      <span class="ml-2 text-sm text-gray-500">
        Saving... | Đang lưu...
      </span>
    }
  `
})
export class FeatureToggleComponent {
  isEnabled = signal(false);
  isLoading = signal(false);

  async handleToggle(checked: boolean) {
    this.isLoading.set(true);

    try {
      // Simulate API call | Mô phỏng gọi API
      await this.saveToServer(checked);
    } finally {
      this.isLoading.set(false);
    }
  }

  async saveToServer(value: boolean): Promise<void> {
    return new Promise(resolve => setTimeout(resolve, 1000));
  }
}
```

## Props | Thuộc Tính

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `checked` | `boolean` | `false` | Switch state <br/> Trạng thái công tắc |
| `status` | `'basic' \| 'primary' \| 'success' \| 'info' \| 'warning' \| 'danger'` | `'primary'` | Color theme <br/> Màu chủ đề |
| `disabled` | `boolean` | `false` | Disable switch <br/> Vô hiệu hóa công tắc |
| `label` | `string` | - | Label text <br/> Văn bản nhãn |

### Outputs | Sự Kiện

| Event | Type | Description |
|-------|------|-------------|
| `checkedChange` | `boolean` | Emitted when switch toggles <br/> Phát ra khi công tắc chuyển đổi |

### Feature Flags | Cờ Tính Năng

```typescript
@Component({
  template: `
    <div class="space-y-4">
      <h3 class="text-xl font-bold">Feature Flags | Cờ Tính Năng</h3>

      <div class="border rounded-lg divide-y dark:border-gray-700">
        @for (feature of features(); track feature.id) {
          <div class="p-4 flex items-center justify-between">
            <div>
              <div class="font-medium">{{ feature.name }}</div>
              <div class="text-sm text-gray-500 dark:text-gray-400">
                {{ feature.description }}
              </div>
            </div>
            <g-switch
              [checked]="feature.enabled"
              (checkedChange)="toggleFeature(feature.id, $event)"
              [status]="feature.status">
            </g-switch>
          </div>
        }
      </div>
    </div>
  `
})
export class FeatureFlagsComponent {
  features = signal([
    {
      id: 'beta-feature',
      name: 'Beta Feature',
      description: 'Enable beta features',
      enabled: false,
      status: 'primary' as const
    },
    {
      id: 'experimental',
      name: 'Experimental Mode',
      description: 'Try experimental features',
      enabled: false,
      status: 'warning' as const
    },
    {
      id: 'debug',
      name: 'Debug Mode',
      description: 'Show debug information',
      enabled: false,
      status: 'danger' as const
    }
  ]);

  toggleFeature(id: string, enabled: boolean) {
    this.features.update(features =>
      features.map(f =>
        f.id === id ? { ...f, enabled } : f
      )
    );
  }
}
```

## Best Practices | Thực Hành Tốt Nhất

1. **Use for binary states** | **Dùng cho trạng thái nhị phân**: Switch is best for on/off, true/false states

2. **Immediate action** | **Hành động ngay lập tức**: Changes should take effect immediately without requiring a save button

3. **Clear labels** | **Nhãn rõ ràng**: Use descriptive labels that indicate what the switch controls

4. **Loading feedback** | **Phản hồi đang tải**: Disable switch and show loading indicator during async operations

5. **Dangerous actions** | **Hành động nguy hiểm**: Use warning/danger status for potentially destructive toggles

6. **Group related switches** | **Nhóm công tắc liên quan**: Organize switches by category in settings
