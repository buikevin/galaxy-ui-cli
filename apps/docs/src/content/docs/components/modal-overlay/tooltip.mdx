The **Tooltip** component displays informative text when users hover over, focus on, or tap an element. Inspired by shadcn's Tooltip component.

**Tooltip** là component hiển thị văn bản thông tin khi người dùng hover, focus, hoặc chạm vào một phần tử. Lấy cảm hứng từ Tooltip component của shadcn.

## Features | Tính Năng

- **12 placement options** | **12 vị trí đặt**: Top, bottom, left, right with start/end variants
- **2 trigger types** | **2 kiểu kích hoạt**: Hover (default), click
- **Delay control** | **Kiểm soát độ trễ**: Configurable show/hide delays
- **Arrow indicator** | **Mũi tên chỉ**: Optional arrow pointing to target
- **Keyboard support** | **Hỗ trợ bàn phím**: Focus trigger for keyboard navigation
- **Dark mode** | **Chế độ tối**: Full dark mode support

## Usage | Sử Dụng

### Basic Usage | Sử Dụng Cơ Bản

```typescript
import { Component } from '@angular/core';
import { TooltipDirective } from '@/components/ui/tooltip';

@Component({
  selector: 'app-example',
  standalone: true,
  imports: [TooltipDirective],
  template: `
    <button gTooltip="This is a tooltip">
      Hover me
    </button>
  `
})
export class ExampleComponent {}
```

### Custom Delay | Độ Trễ Tùy Chỉnh

```typescript
@Component({
  template: `
    <!-- Default delay (200ms show, 0ms hide) -->
    <button gTooltip="Default delay">
      Default
    </button>

    <!-- Custom show delay | Độ trễ hiển thị tùy chỉnh -->
    <button
      gTooltip="Shows after 500ms"
      [tooltipShowDelay]="500">
      Slow Show
    </button>

    <!-- Custom hide delay | Độ trễ ẩn tùy chỉnh -->
    <button
      gTooltip="Hides after 1000ms"
      [tooltipHideDelay]="1000">
      Slow Hide
    </button>

    <!-- No delay | Không có độ trễ -->
    <button
      gTooltip="Instant tooltip"
      [tooltipShowDelay]="0">
      Instant
    </button>
  `
})
```

## Props | Thuộc Tính

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `gTooltip` | `string \| TemplateRef` | required | Tooltip content <br/> Nội dung tooltip |
| `tooltipPlacement` | `'top' \| 'top-start' \| 'top-end' \| 'bottom' \| 'bottom-start' \| 'bottom-end' \| 'left' \| 'left-start' \| 'left-end' \| 'right' \| 'right-start' \| 'right-end'` | `'top'` | Tooltip position <br/> Vị trí tooltip |
| `tooltipTrigger` | `'hover' \| 'click'` | `'hover'` | Trigger type <br/> Kiểu kích hoạt |
| `tooltipShowDelay` | `number` | `200` | Show delay in ms <br/> Độ trễ hiển thị (ms) |
| `tooltipHideDelay` | `number` | `0` | Hide delay in ms <br/> Độ trễ ẩn (ms) |
| `tooltipArrow` | `boolean` | `true` | Show arrow indicator <br/> Hiển thị mũi tên |
| `tooltipDisabled` | `boolean` | `false` | Disable tooltip <br/> Vô hiệu hóa tooltip |

## Accessibility | Khả Năng Tiếp Cận

- Uses ARIA `role="tooltip"` for semantic meaning
- Proper `aria-describedby` linking element to tooltip
- Keyboard accessible (focus trigger)
- Respects `prefers-reduced-motion`
- Screen reader support
- Focus visible indicators

### Disabled Button Explanations | Giải Thích Nút Vô Hiệu Hóa

```typescript
@Component({
  template: `
    <div class="space-x-4">
      <!-- Disabled submit button | Nút gửi bị vô hiệu hóa -->
      <span
        [gTooltip]="getSubmitTooltip()"
        [tooltipDisabled]="canSubmit()">
        <button
          [disabled]="!canSubmit()"
          class="px-4 py-2 bg-blue-600 text-white rounded disabled:opacity-50 disabled:cursor-not-allowed">
          Submit Form
        </button>
      </span>

      <!-- Disabled delete button | Nút xóa bị vô hiệu hóa -->
      <span
        gTooltip="You don't have permission to delete this item"
        [tooltipDisabled]="hasDeletePermission()">
        <button
          [disabled]="!hasDeletePermission()"
          class="px-4 py-2 bg-red-600 text-white rounded disabled:opacity-50 disabled:cursor-not-allowed">
          Delete
        </button>
      </span>
    </div>
  `
})
export class DisabledButtonExample {
  canSubmit = signal(false);
  hasDeletePermission = signal(false);

  getSubmitTooltip(): string {
    const errors: string[] = [];

    if (!this.formValid()) {
      errors.push('Please fill in all required fields');
    }
    if (!this.termsAccepted()) {
      errors.push('Please accept terms and conditions');
    }

    return errors.join('. ');
  }

  formValid = signal(false);
  termsAccepted = signal(false);
}
```

## Best Practices | Thực Hành Tốt Nhất

1. **Keep it brief** | **Giữ ngắn gọn**: Tooltips should be concise, ideally one sentence

2. **Don't repeat visible text** | **Đừng lặp lại văn bản hiển thị**: Add supplementary information, not duplicate content

3. **Use for non-critical info** | **Dùng cho thông tin không quan trọng**: Don't hide essential information in tooltips

4. **Appropriate trigger** | **Kích hoạt phù hợp**: Use hover for quick info, avoid click for mobile-first designs

5. **Accessible fallback** | **Dự phòng truy cập**: Ensure content is accessible without tooltips (aria-label, etc.)
