The **Button** component provides a flexible and accessible way to create interactive buttons with various visual styles inspired by Nebular's design system.

## Features

- Multiple status colors (basic, primary, success, info, warning, danger)
- Five size variants (tiny, small, medium, large, giant)
- Three appearances (filled, outline, ghost)
- Three shape options (rectangle, semi-round, round)
- Full width support
- Disabled state
- Dark mode support

## Usage

### Basic Usage

```typescript
import { Component } from '@angular/core';
import { ButtonComponent } from '@/components/ui/button';

@Component({
  selector: 'app-example',
  standalone: true,
  imports: [ButtonComponent],
  template: `
    <button galaxyButton>Click me</button>
  `
})
export class ExampleComponent {}
```

### Statuses

The button supports multiple status colors to convey different meanings:

```html
<button galaxyButton [buttonStatus]="'basic'">Basic</button>
<button galaxyButton [buttonStatus]="'primary'">Primary</button>
<button galaxyButton [buttonStatus]="'success'">Success</button>
<button galaxyButton [buttonStatus]="'info'">Info</button>
<button galaxyButton [buttonStatus]="'warning'">Warning</button>
<button galaxyButton [buttonStatus]="'danger'">Danger</button>
```

### Sizes

Choose from five different sizes:

```html
<button galaxyButton [buttonSize]="'tiny'">Tiny</button>
<button galaxyButton [buttonSize]="'small'">Small</button>
<button galaxyButton [buttonSize]="'medium'">Medium</button>
<button galaxyButton [buttonSize]="'large'">Large</button>
<button galaxyButton [buttonSize]="'giant'">Giant</button>
```

### Appearances

The button comes in three appearance styles:

```html
<!-- Filled (default) -->
<button galaxyButton [buttonAppearance]="'filled'">Filled</button>

<!-- Outline -->
<button galaxyButton [buttonAppearance]="'outline'">Outline</button>

<!-- Ghost -->
<button galaxyButton [buttonAppearance]="'ghost'">Ghost</button>
```

### Shapes

Customize the button's border radius:

```html
<button galaxyButton [buttonShape]="'rectangle'">Rectangle</button>
<button galaxyButton [buttonShape]="'semi-round'">Semi-round</button>
<button galaxyButton [buttonShape]="'round'">Round</button>
```

### Full Width

Make the button span the full width of its container:

```html
<button galaxyButton [buttonFullWidth]="true">Full Width Button</button>
```

### Disabled State

```html
<button galaxyButton [buttonDisabled]="true">Disabled</button>
```

## Props

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `buttonStatus` | `'basic' \| 'primary' \| 'success' \| 'info' \| 'warning' \| 'danger'` | `'basic'` | The color scheme of the button |
| `buttonSize` | `'tiny' \| 'small' \| 'medium' \| 'large' \| 'giant'` | `'medium'` | The size of the button |
| `buttonAppearance` | `'filled' \| 'outline' \| 'ghost'` | `'filled'` | The visual style of the button |
| `buttonShape` | `'rectangle' \| 'semi-round' \| 'round'` | `'rectangle'` | The border radius style |
| `buttonDisabled` | `boolean` | `false` | Whether the button is disabled |
| `buttonFullWidth` | `boolean` | `false` | Whether the button should span full width |

## Accessibility

- Uses semantic `<button>` element
- Supports keyboard navigation
- Properly handles disabled state with `disabled` attribute
- Focus visible with ring indicator
- High contrast ratios for text readability

## Best Practices

1. **Use appropriate statuses**:
   - `primary` for main actions
   - `danger` for destructive actions
   - `success` for confirmations
   - `basic` for secondary actions

2. **Size consistency**: Use consistent button sizes within the same context

3. **Clear labels**: Use descriptive button text that clearly indicates the action

4. **Loading states**: Disable and show loading indicator during async operations

5. **Accessibility**: Always provide meaningful text, even for icon-only buttons (use `aria-label`)
