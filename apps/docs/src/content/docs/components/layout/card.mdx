The **Card** component provides a beautiful container for grouping related information with optional header, body, and footer sections.

## Features

- Multiple status colors for visual distinction
- Five size variants
- Optional accent border (left border highlight)
- Elevation shadow effects
- Composable header, body, and footer
- Dark mode support

## Installation

```bash
npx galaxy-ui-cli@latest add card
```

## Usage

### Basic Card

```typescript
import { Component } from '@angular/core';
import { CardComponent, CardHeaderComponent, CardBodyComponent, CardFooterComponent } from '@/components/ui/card';

@Component({
  selector: 'app-example',
  standalone: true,
  imports: [CardComponent, CardHeaderComponent, CardBodyComponent, CardFooterComponent],
  template: `
    <galaxy-card>
      <galaxy-card-header>
        <h3>Card Title</h3>
      </galaxy-card-header>
      <galaxy-card-body>
        <p>This is the card content. You can put any content here.</p>
      </galaxy-card-body>
      <galaxy-card-footer>
        <button>Action</button>
      </galaxy-card-footer>
    </galaxy-card>
  `
})
export class ExampleComponent {}
```

### Simple Card (Body Only)

```html
<galaxy-card>
  <galaxy-card-body>
    <p>A simple card with just a body.</p>
  </galaxy-card-body>
</galaxy-card>
```

  <galaxy-card-header>Error Card</galaxy-card-header>

### With Accent Border

Add a colored left border for visual emphasis:

```html
<galaxy-card [cardAccent]="'primary'">
  <galaxy-card-header>Accent Card</galaxy-card-header>
  <galaxy-card-body>
    This card has a primary accent border on the left.
  </galaxy-card-body>
</galaxy-card>
```

### Different Sizes

```html
<galaxy-card [cardSize]="'small'">
  <galaxy-card-body [bodySize]="'small'">
    Small card with less padding
  </galaxy-card-body>
</galaxy-card>

<galaxy-card [cardSize]="'large'">
  <galaxy-card-body [bodySize]="'large'">
    Large card with more padding
  </galaxy-card-body>
</galaxy-card>
```

### Without Elevation

```html
<galaxy-card [cardElevated]="false">
  <galaxy-card-body>
    Flat card without shadow elevation
  </galaxy-card-body>
</galaxy-card>
```

## Props

### Card

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `cardStatus` | `CardStatus` | `'basic'` | Border color theme |
| `cardSize` | `CardSize` | `'medium'` | Overall size (affects children) |
| `cardAccent` | `CardStatus \| null` | `null` | Accent border color |
| `cardElevated` | `boolean` | `true` | Enable shadow elevation |

### Card Header

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `headerSize` | `CardSize` | `'medium'` | Padding size |

### Card Body

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `bodySize` | `CardSize` | `'medium'` | Padding size |

### Card Footer

| Property | Type | Default | Description |
|----------|------|---------|-------------|
| `footerSize` | `CardSize` | `'medium'` | Padding size |

## Best Practices

1. **Use headers for titles**: Always use `galaxy-card-header` for titles to maintain consistent styling

2. **Status colors**: Use status colors to convey meaning:
   - `primary` for important information
   - `success` for positive outcomes
   - `warning` for cautions
   - `danger` for errors

3. **Accent borders**: Use accent borders to highlight key cards without overwhelming the design

4. **Consistent sizing**: Keep card sizes consistent within the same context
